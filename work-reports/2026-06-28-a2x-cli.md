## Subcommands

### `a2x run`
Parse Œ£‚àû program from `--program`, `--stdin`, or `--file`; execute via orchestrator; print result.
- Parses via `a2x_sigma::parse_program()`
- Computes program ID via Blake3 hashing
- Dispatches to `Orchestrator::dispatch()` on CCS VM
- Shows execution result (packet count + serialized packets)

### `a2x parse`
Parse Œ£‚àû input and display packet breakdown.
- `--verbose` flag shows I/C/P/D field-level details (operator debug names, labels, payload size)
- Parses from string, stdin, or file (same `read_input` helper)

### `a2x agents`
List registered agents on the A2X bus.
- Registers 5 built-in demo agents (orch-1, cli-1, cli-2, llm-1, ccs-1)
- Filter by `--type-filter` (orchestrator, cli, llm, ccs, omega, entity) or `--capability` (execute, fs, net, shell, probe, custom)
- Table output: ID, Type, Online (‚óè/‚óã), Capabilities

### `a2x probe`
Inspect an agent's internal state.
- Creates agent of specified type and calls `state_summary()`
- Shows: agent ID, lifecycle state, current program, IP, WorldGraph size, MemoryTrace length, uptime
- Supports all 4 implemented agent types: orchestrator, cli, llm, ccs

## Design Decisions

1. **`anyhow` for CLI error handling** ‚Äî per PLAN.md ¬ß11: "Application crates (`a2x-cli`) use `anyhow::Result`." Library crates use typed errors.
2. **`parse_program` error boxing** ‚Äî `a2x_sigma::parse_program()` returns `Box<dyn Error>` which doesn't implement `Send + Sync`. Wrapped with `anyhow::anyhow!()` for `.context()`-style messages.
3. **`Bus::discover()` instead of pub field** ‚Äî kept `discovery` private to Bus; added a delegating method for the discovery use case.
4. **Built-in demo agents** ‚Äî the `agents` subcommand registers 5 agents for demo purposes. No persistent config file yet (Phase 6).
5. **Test program selection** ‚Äî `test_cmd_run_valid` uses a simple HALT program (`I:‚úï, P:‚úï`) rather than the full sample from PLAN.md, because the sample contains `‚§à` (Descend to `‚ü®sys‚ü©` sub-program) which causes a valid `VmError::UndefinedLabel` in Phase 0.
