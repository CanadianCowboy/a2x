## What Is A2X?

A2X (Agent-to-Anything) is a **three-layer programming language stack** designed
for AI agents:

| Layer | Name | Role |
|:-----:|------|------|
| **Σ∞** | Sigma Infinity | Symbolic ISA — Unicode-dense instruction packets |
| **Ω** | Omega | Compiled latent representation — pure tensors |
| **CCS** | CryoCore Cognitive Substrate | Runtime VM — WorldGraph, StateField, MemoryTrace |

A2X is NOT a protocol (like MCP/A2A), NOT a framework (like LangChain), NOT a
binary format (like WASM). It's a **language** — that AI agents write, compile,
and execute programs in.

## Architecture

```
Σ∞ SOURCE → [Ω COMPILER] → Ω LATENT → [CCS RUNTIME] → Result
                                ↓
                          [A2X BUS] — routing, transport, agent discovery
                                ↓
                          [GATEWAY] — HTTP/WS/TCP listeners, entities, auth
```

## Key Types

### a2x-core
- `AgentId` — string-based agent identifier
- `AgentType` — enum: Orchestrator, Cli, Llm, Ccs, Omega, Entity
- `Capability` — enum: Execute, FileSystem, Network, Shell, Probe, Custom(String)
- `Opcode` — enum: Nop, Bind, Differentiate, Ground, Evolve, Reflect, Plan, Actuate, Jump, Branch, Call, Return, Fork, Merge, Halt, Custom(String)
- `ConceptVector` — dense f32 embedding vector
- `ProgramId` — Blake3 content-addressed program identifier
- `StateSnapshot` — agent state for probing
- `Packet` — unified message type: Raw(Vec<u8>) | Sigma(SigmaPacket) | Omega(OmegaPacket)

### a2x-sigma
- `SigmaPacket` — single instruction: intent + context + plan + data fields
- `SigmaProgram` — sequence of SigmaPackets with labels and sub-programs
- `IntentOp` — enum: Lightning(⚡), Star(✦), Synthesis(✣), Cancel(✕), Split(⩫), Contradiction(⟁), Delay(⧖), Parallel(⥁), Merge(⫴)
- `PlanOp` — enum: Sequential(⥂), Branch(⤐), Descend(⤈), Ascend(⤉), Swarm(⥁), Merge(⤑)
- `DataOp` — enum: Empty(⌵), GraphDelta(⌮), Scalar(⌳), Vector(⌱), Payload(⌴)

### a2x-ccs
- `CcsVm` — the execution VM: WorldGraph + StateField + MemoryTrace + IP
- `VmStatus` — enum: Running, Halted, Yield, Suspended, Fault(VmError)
- `WorldGraph` — petgraph-based concept graph with label index
- `StateField` — flat f32 array with named regions (belief, attention, temporal)
- `MemoryTrace` — execution log of MemoryEntry records

## Conventions

1. **Plan first, code second** — read the relevant sub-plan in `plans/` before writing code
2. **Sub-plan references** — every file references its plan: `// See plans/03-ccs-vm.md §5`
3. **One concept per file** — module `mod.rs` re-exports only
4. **Feature gates** — optional deps behind named features; `serde` gate on serializable types
5. **No unwrap in library code** — use `Result<T, E>` or `Option<T>` with proper error handling
6. **Tests for all changes** — infrastructure gets unit tests; cognitive code gets property tests
7. **Work reports required** — before committing, create `work-reports/YYYY-MM-DD-description.md`

## Common Tasks

### Adding a new Σ∞ operator
1. Add the operator symbol to `a2x-sigma/src/intent.rs` (or plan.rs / data.rs)
2. Add the Opcode mapping to `a2x-ccs/src/vm.rs::map_to_opcode()`
3. Add the operator implementation in `a2x-ccs/src/operators/`
4. Wire it into `CcsVm::step()` dispatch match
5. Add tests

### Adding a new agent type
1. Add variant to `AgentType` in `a2x-core/src/agent_id.rs`
2. Create `a2x-agents/src/new_agent.rs` implementing `Agent` trait
3. Register in `Gateway::register_builtin_agents()` and `a2x-cli/src/main.rs`
4. Add tests

### Adding a new dependency
1. Add to `[workspace.dependencies]` in root `Cargo.toml`
2. Add to the specific crate's `Cargo.toml` with `{ workspace = true }`
3. Never add to `a2x-core` (zero-dependency crate)
